# Muse 快速开始指南

## 🚀 5分钟快速上手

### 1. 安装插件（30秒）

```bash
# 图标已自动生成，无需额外操作
```

1. 打开 Chrome 浏览器
2. 访问 `chrome://extensions/`
3. 开启"开发者模式"
4. 点击"加载已解压的扩展程序"
5. 选择 `muse` 文件夹
6. ✅ 安装完成！

### 2. 打开编辑器（10秒）

**方式一：使用完整编辑器（推荐）**
1. 点击浏览器工具栏的 Muse 图标（蓝色 M）
2. 点击弹窗顶部的 **🚀 打开编辑器** 按钮
3. 编辑器在新标签页中打开
4. ✅ 完全支持中文输入！

**方式二：使用弹窗**
1. 点击浏览器工具栏的 Muse 图标
2. 直接在弹窗中编辑
3. ⚠️ 可能无法使用输入法

### 3. 创建第一个文档（2分钟）

1. 在文件名输入框中输入：`笔记.md`
2. 在编辑器中输入内容：

```markdown
# 我的第一篇笔记

这是一个测试文档。

## 待办事项
- [ ] 学习 Markdown
- [ ] 使用 Muse 编辑器
- [ ] 提交第一个 Git 版本
```

3. 点击 **💾 保存** 按钮
4. ✅ 文件保存成功！

### 4. 初始化 Git（30秒）

1. 点击 **🔧 Init** 按钮
2. 看到提示"Git 仓库已初始化"
3. ✅ Git 初始化完成！

### 5. 提交第一个版本（30秒）

1. 在"提交信息"输入框中输入：`初始版本：创建笔记`
2. 点击 **✅ Commit** 按钮
3. 看到提示"已提交：初始版本：创建笔记"
4. ✅ 第一个版本提交成功！

### 6. 查看提交历史（10秒）

1. 点击 **📜 Log** 按钮
2. 弹出面板显示提交历史
3. 可以看到刚才的提交记录
4. ✅ 查看历史成功！

### 7. 启用预览（10秒）

1. 点击 **👁️ 预览** 按钮
2. 编辑器分为左右两栏
3. 左侧编辑，右侧实时预览
4. ✅ 预览模式启用！

## 🎯 核心功能速查

| 按钮 | 功能 | 说明 |
|------|------|------|
| 🚀 打开编辑器 | 打开完整编辑器 | 在新标签页打开，支持输入法 |
| 📄 新建 | 新建文件 | 清空当前内容 |
| 💾 保存 | 保存文件 | 保存到 Chrome Storage |
| 📂 打开 | 打开文件 | 从文件列表中选择 |
| 👁️ 预览 | 切换预览 | 分屏显示 Markdown 预览 |
| 🔧 Init | 初始化 Git | 首次使用需要初始化 |
| ✅ Commit | 提交更改 | 提交到 Git 历史 |
| 📜 Log | 查看历史 | 显示所有提交记录 |
| 🔍 Diff | 查看差异 | 对比当前版本与上次提交 |

## ⚡ 常用操作流程

### 创建和保存文档
```
输入文件名 → 编写内容 → 点击保存
```

### Git 版本控制
```
初始化 Git → 编辑文档 → 保存 → 输入提交信息 → 提交
```

### 查看历史版本
```
点击 Log → 查看所有提交 → 关闭面板
```

### 对比差异
```
编辑文档 → 点击 Diff → 查看变更内容
```

## 💡 使用技巧

### 1. 推荐使用完整编辑器
- 点击 **🚀 打开编辑器** 按钮
- 完全支持中文、日文、韩文输入
- 更大的编辑空间
- 更好的使用体验

### 2. 定期提交版本
- 编辑一段时间后提交一次
- 写清楚提交信息
- 方便追溯历史版本

### 3. 使用预览功能
- 实时查看 Markdown 渲染效果
- 确保格式正确

### 4. 管理多个文件
- 使用文件列表管理多个文档
- 每个文档独立保存
- 共享同一个 Git 历史

## ❓ 常见问题

### Q: 无法输入中文怎么办？
A: 点击 **🚀 打开编辑器** 按钮，在新标签页中使用完整编辑器。

### Q: 数据保存在哪里？
A: 所有数据保存在 Chrome 浏览器的本地存储中，不会上传到服务器。

### Q: 卸载插件会丢失数据吗？
A: 会的，卸载前请导出重要文件（复制内容）。

### Q: 可以导出文件吗？
A: 当前版本可以复制内容后粘贴到其他地方。未来版本将添加导出功能。

### Q: Git 提交能恢复到历史版本吗？
A: 当前版本只能查看历史，不能恢复。未来版本将添加恢复功能。
