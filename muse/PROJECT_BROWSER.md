# Muse 项目浏览器使用指南

## 🎉 全新界面

Muse 现在提供全新的项目浏览器界面！弹窗不再是编辑器，而是一个强大的项目和文件管理器。

## 📱 弹窗界面（项目浏览器）

### 主要功能

#### 1. 搜索栏
- 位于顶部，可以搜索项目名称和文件名
- 实时过滤显示匹配的项目和文件
- 输入关键词即可快速定位

#### 2. 项目列表
- 以卡片形式展示所有项目
- 点击项目可展开/收起查看文件列表
- 每个项目显示：
  - 📁 项目图标
  - 项目名称
  - 项目描述（如果有）
  - 文件数量

#### 3. 文件列表
- 展开项目后显示该项目下的所有文件
- 显示文件名和最后修改时间
- 点击文件直接在编辑器中打开

### 操作按钮

#### 顶部工具栏
- **📁 新建项目** - 创建新的项目
- **📄 新建文件** - 在指定项目下创建新文件
- **🚀 打开编辑器** - 在新标签页打开完整编辑器

#### 项目操作（鼠标悬停显示）
- **✏️ 重命名** - 重命名项目
- **🗑️ 删除** - 删除项目及其所有文件

#### 文件操作（鼠标悬停显示）
- **✏️ 重命名** - 重命名文件
- **🗑️ 删除** - 删除文件

#### 状态栏
- 显示项目总数
- 显示文件总数

## 💻 完整编辑器

### 打开方式

1. **从项目浏览器**
   - 点击文件 → 自动在新标签页打开编辑器
   - 点击"🚀 打开编辑器"按钮 → 打开空白编辑器

2. **直接访问**
   - 在浏览器中输入：`chrome-extension://[扩展ID]/editor.html`

### 编辑器功能

- ✅ 完整的 Markdown 编辑
- ✅ 实时预览（分屏）
- ✅ 文件保存
- ✅ Git 版本控制
- ✅ 完全支持输入法（中文、日文、韩文）

## 📂 使用流程

### 快速开始

1. **创建第一个项目**
   ```
   点击浏览器工具栏 Muse 图标
   → 点击"📁 新建项目"
   → 输入项目名称（如：工作笔记）
   → 输入项目描述（可选）
   → 点击"创建"
   ```

2. **创建文件**
   ```
   点击"📄 新建文件"
   → 选择项目
   → 输入文件名（如：会议记录.md）
   → 点击"创建"
   → 自动在编辑器中打开
   ```

3. **编辑文件**
   ```
   在编辑器中输入 Markdown 内容
   → 点击"💾 保存"
   → 文件自动保存到项目
   ```

4. **浏览文件**
   ```
   点击 Muse 图标打开项目浏览器
   → 点击项目展开文件列表
   → 点击文件名打开编辑
   ```

## 🔍 搜索功能

### 搜索项目

在搜索框输入项目名称的关键词：
- 输入"工作" → 显示所有包含"工作"的项目
- 输入"学习" → 显示所有包含"学习"的项目

### 搜索文件

在搜索框输入文件名关键词：
- 输入"React" → 显示所有包含"React"的文件
- 输入"会议" → 显示所有包含"会议"的文件

### 组合搜索

搜索会同时匹配项目名称和文件名，显示所有匹配项。

- **项目卡片** - 白色背景，悬停时显示阴影
- **展开状态** - 展开的项目有蓝色边框
- **文件列表** - 浅灰色背景区分层级

- 点击项目头部 → 展开/收起动画
- 悬停项目/文件 → 显示操作按钮
- 点击文件 → 在新标签页打开编辑器
- 悬停按钮 → 颜色变化和阴影效果

## ❓ 常见问题

### Q: 项目浏览器在哪里？
A: 点击浏览器工具栏的 Muse 图标即可打开项目浏览器弹窗。

### Q: 如何编辑文件？
A: 在项目浏览器中点击文件名，会在新标签页打开编辑器。

### Q: 可以在弹窗中编辑吗？
A: 不可以。新版本中弹窗专门用于浏览和管理项目文件，编辑功能在独立的标签页中。

### Q: 为什么要分开项目浏览器和编辑器？
A: 因为弹窗环境无法正常使用输入法，将编辑器放在独立标签页可以完美支持中文输入。

### Q: 旧版本的文件怎么办？
A: 系统会自动迁移到"默认项目"，不会丢失数据。

### Q: 可以在项目之间移动文件吗？
A: 当前版本暂不支持，需要手动复制粘贴内容。未来版本会添加此功能。

### Q: 最多可以创建多少个项目？
A: 理论上没有限制，但受 Chrome Storage 容量限制（约 10MB）。

### Q: 搜索支持正则表达式吗？
A: 当前版本只支持简单的文本匹配，不支持正则表达式。
